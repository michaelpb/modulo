Zero set-up - Like the golden age of jQuery, just drop it into any HTML file. Unlike jQuery, that 1 file packs in all the creature-comforts of a modern component framework.

Very vanilla - Modulo doesn't re-invent any wheels. Like Polymer (litElement) or Skate.js, Modulo is built as a thin-layer over the custom element web standard that is now well-supported. Modulo's templates use a common syntax, which means your editor can probably already syntax highlight Modulo template code!

Progressive enhancement - JAMStack is fine and all, but what about all our existing server-side apps in Ruby on Rails, Python and Django, or that one home-grown PHP monster that will never go away? Modulo was made to give structure and convenience to that extra sprinkle of JavaScript that will make your "traditional" web apps shine.

CParts - Create components by putting together one or more Component Parts or CParts. Mix Script, Style, State, Props, and Templates to make components that have one or all, or multiples, of these features. Combine CParts between components for a powerful system of "composition-based" inheritance that's more familiar to web developers.

Everything is swappable - Components configure themselves, meaning you can "mix and match" templating languages, or try out DOM-resolution on or off per-component basis.

Lifecycle hooks and directives - Hook your own code into the component rendering lifecycle, either through callbacks, or by defining custom attribute-based directives.

"It's like React" - Modulo is batteries-included to be like React.js or Vue.js, except without the complications of dependencies or build systems. You get props, state, lifecycle methods, smart DOM resolution, encapsulated components, and a simple namespace and library loading system.

Clean & concise - The code-base is intentionally kept short & sweet: Under 1000 SLOC, 80 char lines, 4 spaces of indent, and a low-complexity imperative style. When something goes wrong, git blame Modulo.

A basis for whatever comes next - Want to write your own component framework, but don't want to start from scratch? Read the "literate" code for a starting spot, and fork away!
