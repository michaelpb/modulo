Zero set-up - Like the golden age of jQuery, just drop it into any HTML file. Unlike jQuery, that 1 file packs in many of the creature-comforts of a modern component framework, without any dependencies or new syntax to learn.

A gentle learning curve - Modulo's declarative, synchronous coding style is intended to be easier for newbie coders to pick-up—in fact, many components can be developed without use of any JavaScript!

Progressive Web Apps Progressive Enhancement - Modulo makes few assumptions, and renders components mixed with plain HTML for easier integration into backend apps. JAMStack and PWA's are fine and all, but what about all our existing server-side apps in Ruby on Rails, Python, Go, or that one home-grown PHP monster that will never go away? Sprinkle in some Modulo components to make your "traditional" web apps shine.

CParts - CParts are like Lego blocks for your components! Create components by snapping together one or more Component Parts or CParts. Mix Script, Style, State, Props, and Templates to make components that have one or all, or even multiples, of these features. Reusing CParts between components provides a powerful system of "composition-based" inheritance.

Everything is swappable - Components configure themselves, meaning you can create custom CParts, "mix and match" your favorite JS templating languages, or fine-tune DOM-resolution behavior on a per-component basis.

Lifecycle hooks and directives - Hook your own code into the rendering process, either through lifecycle callbacks, or by defining custom attribute-based directives.

"It's like React" - Modulo is "batteries-included" to function like React.js or Vue.js. You get encapsulated components with state management, two-way data binding, props & types, computed values, DOM resolution, event directives, and more. You even get a simple importing and namespacing system, that self-packs with your components into a single JS file!

Clean & concise - The code-base of Modulo itself is intentionally kept short & sweet: It hovers around roughly 1000 SLOC, 80 char line limit, 4 spaces of indent, and a low-complexity imperative style. When something goes wrong (which it probably will since it's a new framework), it's easy to "git blame" Modulo.

A basis for whatever comes next - Want to write your own component framework? Modulo's core is in only a few hundred lines of code. Read the source code, fork away, and swap the "batteries-included" for your own!
