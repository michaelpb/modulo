Design goals of Modulo

HTML-first

One important design goal of Modulo is to be much more HTML-oriented compared to other web framework. In some ways, it's could be thought of as more of an HTML framework written in JavaScript than a JavaScript framework written in JavaScript.

  1. HTML-first syntax

    Modulo is HTML-first in that it piggy-backs off of HTML syntax. CParts are declared using HTML tags, within other HTML tags, as opposed to within JavaScript. Script tags embed JS into HTML, instead of the other way around (e.g. JSX in React).

  2. HTML-first workflow

    Secondly, it is HTML-first in terms of development workflow. A greenfield Modulo-powered website starts as a static site of HTML and CSS files, created and edited by hand. As requirements become more demanding, the vanilla HTML tags are gradually swapped for Modulo web-components, with templated HTML and bundled CSS. Only as requirements grow beyond the common use-cases offered by "canned" CParts does JavaScript come into play.

  3. HTML-first enhancement

    Finally it is HTML-first in terms of interoperability. Modulo exposes itself as a extension to vanilla HTML via the Web Components standard. This means it will comfortably integrate with anything that deals with HTML (PHP, Django, Rails, other frontend frameworks, etc).
