No assumptions

Zero set-up - Like the golden age of jQuery, just drop it in any HTML file. Unlike jQuery, that 1 file packs in a component system, hot-reloading, a minimal build system to pack CSS and JS, and novel Dev Tools based debugging.

Concise - The code-base is intentionally kept short & sweet, so when something goes wrong you can always git blame Modulo

Web Components - Like Polymer, litHtml, or Skate.js, Modulo is built as a thin-layer over existing web standards. Modulo doesn't re-invent any wheels, and instead can leverage existing VDOM reconciliation and template libraries.

CParts

☑ Modern creature-comforts - Dev toolbar with hot-reloading & one-click generation of JS and CSS

Modulo is component-based

☑ State management - The entire framework is only about 1000 lines of code

☑ Component Parts - Create components with CParts, a novel system where script, state, props, and templates can be mixed to make new components.

☑ Extendable life-cycle - Middleware system with lifecycle methods, familiar to React devs
